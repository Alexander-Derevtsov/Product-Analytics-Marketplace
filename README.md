# Анализ продуктовых метрик маркетплейса

## Описание проекта

В рамках проекта проведен комплексный анализ пользовательского поведения маркетплейса с целью оценки удержания пользователей, проверки наличия Product/Market Fit и поиска точек роста продукта.

В ходе исследования были рассчитаны ключевые продуктовые метрики, проведен когортный анализ, выполнена приоритизация продуктовых гипотез с использованием фреймворка ICE и предложен дизайн A/B-теста для проверки наиболее перспективной гипотезы.

---

## Цель проекта

Определить причины стагнации выручки маркетплейса и сформулировать рекомендации, направленные на повышение прибыльности продукта.

---

## Задачи исследования

- провести исследовательский анализ данных;
- рассчитать месячный retention с помощью когортного анализа;
- оценить наличие Product/Market Fit;
- определить ключевые продуктовые метрики;
- выбрать наиболее перспективную гипотезу с помощью ICE Framework;
- определить целевые, прокси- и guardrail-метрики для будущего A/B-теста;
- сформулировать рекомендации для продуктовой команды.

---

## Используемые данные

Использован датасет **Brazilian E-commerce Public Dataset (Olist)**.

Основные таблицы:

- пользователи;
- заказы;
- товарные позиции в заказах.

## Ход исследования

### 1. Предобработка данных

- объединение таблиц;
- обработка пропусков;
- выбор релевантных статусов заказов;
- подготовка данных для анализа.

### 2. Когортный анализ

Для пользователей были построены месячные когорты по дате первого заказа.

Рассчитан месячный retention и построена тепловая карта удержания пользователей.

*Здесь можно вставить изображение retention_heatmap.png.*

---

### 3. Оценка Product/Market Fit

На основании когортного анализа оценено наличие Product/Market Fit и определены основные ограничения роста продукта.

*Здесь можно вставить график retention.*

---

### 4. Анализ продуктовых метрик

Рассчитаны и проанализированы следующие показатели:

- GMV;
- количество активных покупателей;
- Retention;
- среднее количество заказов на пользователя;
- средняя выручка на пользователя.

Для каждой метрики построены графики динамики.

---

### 5. Приоритизация гипотез

Три продуктовые гипотезы были оценены с использованием ICE Framework.

По результатам расчета выбрана гипотеза с наибольшим ожидаемым эффектом.

---

### 6. Проектирование A/B-теста

Для выбранной гипотезы определены:

- целевые метрики;
- прокси-метрики;
- guardrail-метрики.

---

## Основные результаты

- рассчитан месячный retention пользователей;
- оценено наличие Product/Market Fit;
- определены ключевые продуктовые метрики маркетплейса;
- выбрана приоритетная продуктовая гипотеза;
- разработан план оценки эффективности изменений с помощью A/B-тестирования.

---

## Выводы

Исследование показало, что основной точкой роста продукта является увеличение доли повторных заказов. Для этого рекомендуется сосредоточиться на повышении удержания пользователей, снижении количества отмен заказов и улучшении пользовательского опыта после первой покупки.

Предложенные рекомендации могут использоваться продуктовой командой при планировании дальнейшего развития маркетплейса.
